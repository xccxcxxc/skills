# 排版成书

CRITICAL: 本步骤用于“粗合并/标题分类”之后，把 `merge-result/0.rough.md` 处理成最终成书 Markdown。若标题层级未整理，先执行 `references/标题分类.md`。

## 输入/输出
- 输入：`merge-result/0.rough.md`
- 输出：`<书名>.md`
- 可选中间目录：`merge-result/`、`typeset-result/`

## 脚本
```bash
python scripts/typeset_book.py --base-dir "书籍目录"
```

## 规则
1. 先保证标题分类正确；脚本不会替你把章节升成导航 H1。
2. 脚本内部会临时按一级标题 `# ` 切块，处理硬断行与注释空行，再按原顺序写出最终文件。
3. 二级及以下标题不会作为独立成书文件的边界。
4. 若需要调试中间文件，才加 `--keep-intermediate`。

### 原书插图处理（必须判断后执行）
1. 重要的封面、封底、书影、图版和正文配图：复制到 `assets/`，并在 Markdown 中用真实图片引用替换 `🀄️页码.jpg`。
2. **封面只允许出现一次**：
   - 真正的 EPUB 封面用 `assets/cover.jpg` + 导出时的 `--epub-cover-image`；
   - **正文 Markdown 中不要再嵌入同一张封面图**；
   - 也不要同时嵌入封面扫描页 + 版权装饰页当“第二封面”。
3. **必须删除占位符且不要嵌入**：纯目录页、纯章节扉页、章节标题跨页、无正文信息的装饰整页、空白页，以及仅用于版式分隔的图像。
4. 正文中的真实插图、地图、图版才保留。

## 检查
- 最终 Markdown 中：
  - 应进目录的章节都是 `# `；
  - 没有重复封面图片引用；
  - `🀄️` 占位符清零，或仅保留已确认要保留的图。
- 标题层级没有错误。
- 若 typeset 把作者行和图片挤在同一行，导出前手动拆开。
