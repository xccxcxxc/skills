CRITICAL: 按顺序执行以下阶段，勿跳步。

## 输入/输出
- 输入：书籍目录下与书籍同名的 `.md` 文件（例如 `书籍目录/书籍名.md`）。
- 输出：按段落完整切分为多个文件，写入 `书籍目录/translate-split/`（不存在则新建）。

## 脚本参考

- 使用 `scripts/translate_split.py` 完成分割。
- 默认按当前工作目录推导路径：
  - 输入：`<当前目录>/<书籍名>.md`（书籍名默认取当前目录名）
  - 输出：`<当前目录>/translate-split/`
- 可选参数：
  - `--base-dir` 指定书籍目录
  - `--input-file` 指定完整输入文件路径
  - `--book-name` 指定书籍名
  - `--output-dir` 指定完整输出目录
  - `--max-chars` 指定每个文件最大字符数（默认 40000）
  - `--base-dir-from` 使用 UTF-8 文本文件提供书籍目录（首个非空行）
  - `--input-file-from` 使用 UTF-8 文本文件提供输入文件路径（首个非空行）
  - `--book-name-from` 使用 UTF-8 文本文件提供书籍名（首个非空行）
  - `--output-dir-from` 使用 UTF-8 文本文件提供输出目录（首个非空行）

示例：

```bash
python skills/pdf-set/scripts/translate_split.py --base-dir "C:\path\to\book"
```

## 阶段 1：段落切分依据
1. 仅把**空行**（`\n\n`）作为段落分割。
2. 单个回车分隔的行可能仍属同一段，禁止擅自拆段。

## 阶段 2：分割策略
- 任何输出文件**不得**切断一个段落。
- 在满足完整段落的前提下，尽量让每个文件接近 40000 字符上限。
- 如果单个段落超过 40000 字符，只能单独成文件并保留完整内容。

## 阶段 3：输出检查
- 输出目录为 `translate-split/`。
- 文件按顺序编号（`1.md`、`2.md`……），必须从 1 开始，顺序与原文一致。
- 每个文件字符数 < 40000（除非单段超长）。
