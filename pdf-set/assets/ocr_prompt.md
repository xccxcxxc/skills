## 正文字符边界（核心）：识别到什么就输出什么；完全尊重图片原始文字边界。不改写、不补全、不润色、不推断
- 每张图片都要明确识别正文的第一个字符与最后一个字符（包含断句残句）。
- .md 必须从起始字符写到结束字符，不补不删。
- 严禁：猜测、推断、补全句意。
- 例：若 图片内容的 的正文结束字符是「…身体的延」；
  则 输出的文本 必须以「身体的延」结尾；不得以猜想它后面可能是「续」或「伸」等字而补全。
- 例：若图片内容的最后是「。无」，则 输出的文本 必须以「。无」结尾；不得以把单独的「无」删掉。
  若图片的开始是以「中一群人……」你不得认为「中」是一个多余字符而删掉它。
- 结尾为「，」「、」「：」等明显未完结符号时：保持原样输出，不得续写。
- 一页结尾是像「s'atrp-」这样的有未完符号时，输出不可以漏掉最后的-符号。

## 清理（必须删除）
- 页眉页脚（包括小标题，书目，页码）
- 边码/边栏编号；正文中的verses节号
- 不在正文中（和正文有距离），而是在每页页眉中的章节标题名

## OCR 识别规则
- 无文字内容的空页，你就输出🈳+文件名就好了。
- 字符忠实：简体保持简体，繁体保持繁体；不做简繁转换；不换语言（不要把「他/和/的/了」换成外语词）。
- 版式剔除：删除页眉/页脚页码、边码/边栏数字、每页重复的章节标题。
- 文本样式：加粗/斜体/特殊样式一律还原为纯文本。
- 复杂图形处理：
  - 简单数学/希腊字母可用 LaTeX，例如：($S(\\cancel{A})$), \\sigma, $p^o$。（对单独出现的$符号要转义处理以免被识别为公式）
  - **表格优先文本化**（含「正文+表格」混排页）：只要行列结构可辨，就输出 GFM 管道表格，**禁止**因页内有表格就把整页改成 🀄️。
  - 仅当内容**无法文本化**时才插入 🀄️+文件名：纯图示/照片/地图/图版、复杂公式图、扫描损坏到无法辨认行列的表、装饰性整页图。
  - 禁止使用 Markdown 图片语法（`![](...)`）；图片/图版一律用 🀄️+文件名 标记。

### 表格规则（版式贴原表 · 强制不换行 · 紧凑）
目标：**行列结构与原表一致**；单元格文字与原表一致；**单元格内绝不换行**；**表内绝不塞注释**；尽量在一页宽度内完整显示（靠缩小字号与紧凑结构，不靠把表头拆行）。

#### 结构与阅读顺序
- 同一页可「正文 → 表题 → 表格 → 资料来源/表注 → 正文」；顺序与原页一致。
- 表题（如「表5—5 ……」）写在表格**上方**独立段落，不要放进表头单元格。
- 表前、表后各 1 个空行；**表内禁止空行**。
- **表格行不要首行缩进**。

#### 单元格文字（必须）
- 只输出格子里看得见的字/数；空格保持空。
- **禁止**把脚注/尾注/边注正文写进任何单元格（含表头）。
- 表内注号（①②* 等）**只保留符号**，如 `时间差②`；注释移到表外 `<sup>【】</sup>`。
- **单元格内禁止换行 / `<br>` / 软换行**；原格若多行，用空格收成**一行**（显示层会 white-space:nowrap）。
- 百分比、数字、区间照原表；原表无 `%` 不要加。

#### 何时用 GFM，何时用 HTML（关键）
**A. 单层表头（如「表5—6」姓名/职务/…）→ GFM 管道表**
- 表头恰好一行 + 分隔行 + 数据行；每行 `|` 开头结尾。
- 分隔行 `| --- |`；数字列可用 `| :---: |`。
- 列数整表一致。

**B. 多级表头 / 跨列合并（如「表5—5」上层「时间差|单位分布|轨迹类型」，下层「0-5|6-10|…」）→ 必须用 HTML `<table>`**
- **禁止**把多级表头压成一行后丢失上层分组（那会与原表不符）。
- 用 `<thead>` 还原多级表头：`rowspan` / `colspan` 与原表合并方式一致。
- 用 `<tbody>` 写数据行；空单元格写 `<td></td>`。
- 外层：`<div class="table-wrap">...</div>`；**统计密表/多级表头**给 table 加 `class="table-dense"`（强制单元格不换行、略缩小字号）。
- **叙述型名单表**（如「表5—6」姓名/职务/轨迹等长文本列）**不要**加 `table-dense`，用 GFM 管道表，让长单元格正常换行、保持可读字号。
- **每个 th/td 内只写一行纯文本**，不要在格子里再套 `<p>` 或换行。
- 不要用 Markdown 管道表去「模拟」多级表头。
- HTML 保持紧凑：一行一个 `<tr>…</tr>`，勿插入表内空行。

**C. 仅一行表头但列极多（如年龄分布多档）**
- 优先 GFM；表头用最底层列名（`26-30`…）。若原表上层分组必须保留，改用 HTML 两级表头。

#### 紧凑（禁止拉长）
- 不为对齐填充超长空格。
- 不把一张表拆成多张。
- **不靠表头换行省宽度**——显示层强制 nowrap；结构上保持与原表相同的行列/合并。

#### 跨页表格
- 本页有几行输出几行；禁止补全未出现在本页的行。
- 起表页：表题（若有）+ 完整表头（GFM 或 HTML 与结构匹配）+ 本页数据。
- 续表页：若有「续表」等表题则照抄；**必须重复与起表页相同的表头结构**（列名/colspan/列序一致），再接本页新数据行；不重复上页数据。
- 「资料来源」只在原页有则输出。

## 排版规则
### 分段
- 正文段落之间禁止使用单个换行符（`\n`）作为段落分隔符；段落之间必须使用两个连续换行符（`\n\n`），即保留**一个**空行。
- **例外：Markdown 表格**——表内相邻行只用单个换行符连接，表前表后用空行与正文分隔。
- 每页输出结尾不留空行。
- 标题：章节标题或小标题用二级标题 ##；禁止一级/三级标题(目录中的标题不用加##)

### 首行缩进：
  - 以内容的连贯性和排版的空行位置判断该段落是否是新段落，新段段首加两个空格首行缩进。
  - 如果是一连串注释或者正文中列举，或者类似于参考资料中的人名或参考书目列举，你统一给每个不同内容之间都加新行，并且做首行缩进。
  - 若为上一页断段续接（通常无首行缩进且句意未完），则不要缩进。
  - 对话体/诗歌/列表/特殊格式……：每人对话间留空行，每新行都缩进。
  - 不在正文内，单独成段的 🀄️+文件名 或者 单独成段的一对 sup 标签 也需要首行缩进
  - **表格行不要首行缩进**

示例：

```
……上一段落结尾

  下一段落开头
```
### 注释

#### 处理边注 Marginal Notes：括号包裹
- 对 marginal notes（边注）：（可能是穿插在正文中的小字注释或括号内解释）你统一格式化为（注：xxx）的格式并放在正文位置之中。
- 你需要严格判断字型大小来识别正文和边注的区别：大字是正文，小字是边注；
-如果该页结尾有明显未完的边注内容，你只需要留存一个左括号即可，不需要补全右括号；同理，如果该页开头有明显未完的边注内容，你只需要留存一个右括号即可，不需要补全左括号。 

#### 处理尾注 Endnote：不转移
若页脚无注释内容但是正文有标号──则这是尾注 Endnotes类型的书籍；通常会在章节正文结束后或全书的末尾集中列出注释内容；会有 Note 或者 Annotations 之类的小标题提示以下内容为注释内容；你需要：
  - 不删除文内注号；
  - 文内注号改为 <sup>【原注号】</sup> 形式；不加入任何注释内容。
  - 如果没有Note 或者 Annotations 之类的小标题提示，你加一个小标题 ## Note 或者 ## Annotations 来标明注释内容的开始。

处理章节末尾注的标准：完全当作正文内容处理：
  - 尾注内容保持原样，不做内嵌或清理。
  - 注释间保留空行。
  - 不需要sup标签
  - 不需要使用⬆️和⬇️的标志。

#### 处理脚注 Footnote ：转移
- 若页面页脚有对应于正文注号的注释即为脚注Footnote：把脚注文本移到正文中对应注号后，用 <sup>【】</sup> 包裹，并删除原注号与页脚脚注。
- 注释不得分段；原图注释如分段，需合并为一段。(见后例)
- 若注释末尾为「接下页」「-」「接下页注」或明显未完：在注释末尾加 ⬇️。
- 若页脚出现无标号且像是续上页的注释（可能有「续上页/续上页注/接上页注」或无注号）;
  或者是出现了注释，但是对应的标号不在本页（可能在上一页）；
  你需要：
  放在正文最前，用下面方法分隔(注意：空行、⬆️、首行缩进的两个空格)：
```
  <sup>⬆️……</sup>

```
