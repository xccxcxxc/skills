# 排版成书

CRITICAL: 本步骤用于“粗合并”之后，把 `merge-result/0.rough.md` 直接处理成最终成书 Markdown。若标题层级未整理，先执行 `references/标题分类.md`。

## 输入/输出

- 输入：`merge-result/0.rough.md`
- 最终输出：`书籍目录/书籍名.md`
- 默认不保留中间分割文件或排版文件。

## 脚本

使用 `scripts/typeset_book.py`。

默认命令：

```bash
python3 skills/pdf-set/scripts/typeset_book.py --base-dir "书籍名"
```

Windows 示例：

```powershell
python "C:\Git\playground\skills\pdf-set\scripts\typeset_book.py" --base-dir "C:\Git\playground\书籍名"
```

## 规则

1. 对用户而言只有一个阶段：排版成书。
2. 脚本内部会临时按一级标题 `# ` 切块，处理硬断行与注释空行，再按原顺序写出最终文件。
3. 二级及以下标题不会作为独立成书文件的边界。
4. 默认使用临时目录，运行后只保留最终 `书籍名.md`。
5. 若需要调试中间文件，才加 `--keep-intermediate`；此时会写入 `merge-result/1.标题.md` 与 `typeset-result/1.标题.md` 等文件。

### 原书插图处理（必须判断后执行）

1. OCR 中的 `🀄️页码.jpg` 是图片占位符；排版完成后，必须逐页判断其实际内容，不能一律保留或删除。
2. **必须嵌入**：原书封面、封底、书影/版权页中具有视觉价值的图像、彩色/黑白图版页、插图、地图、图表，以及正文理解所必需的配图。将对应原图复制到书籍目录的 `assets/`，并将占位符替换为 Markdown 图片语法，例如：`![封面](assets/1.jpg)`。
3. **必须删除占位符且不要嵌入**：纯目录页、纯章节扉页、章节标题跨页、无正文信息的装饰整页、空白页，以及仅用于版式分隔的图像。
4. 图版如果连续跨多页，逐页保留；图片旁的原有图注文字也应保留。
5. 导出 EPUB 前确认 Markdown 中不存在未处理的 `🀄️页码.jpg`；并确认所有 `assets/` 图片均能被引用。

## 检查

运行后至少确认：

- 最终文件 `书籍目录/书籍名.md` 存在。
- 关键层级没有错误，例如 `# 耶利米書` 下是 `## 一章`，不是 `# 一章`。
