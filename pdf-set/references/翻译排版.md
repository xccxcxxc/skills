CRITICAL: 按顺序执行以下阶段，勿跳步。

## 输入/输出
- 输入：`translate-split/` 中的全部 `.md` 文件。
- 输出：写入 `translate-typeset/`（不存在则新建），文件名与输入文件相同。

## 脚本参考

- 使用 `scripts/translate_typeset.py` 完成排版。
- 默认按当前工作目录推导路径：
  - 输入：`<当前目录>/translate-split/`
  - 输出：`<当前目录>/translate-typeset/`
- 可选参数：
  - `--base-dir` 指定书籍目录
  - `--input-dir` 指定完整输入目录
  - `--output-dir` 指定完整输出目录
  - `--max-chars` 指定每段目标字符数（默认 250）
  - `--buffer-chars` 指定缓冲字符数（默认 200，段落长度 ≤ 目标值+缓冲值不分割）
  - `--base-dir-from` 使用 UTF-8 文本文件提供书籍目录（首个非空行）
  - `--input-dir-from` 使用 UTF-8 文本文件提供输入目录（首个非空行）
  - `--output-dir-from` 使用 UTF-8 文本文件提供输出目录（首个非空行）

示例：

```bash
python .agent/skills/pdf-set/scripts/translate_typeset.py --base-dir "C:\path\to\book"
```

## 阶段 1：段落识别
1. 仅把**空行**（`\n\n`）作为段落分割。
2. 单个回车分隔的行可能仍属同一段，禁止擅自拆段。

## 阶段 2：段落断开规则
1. 检查每个段落的字符数。
2. 如果段落字符数 > 目标值 + 缓冲值（默认 450），则必须断开成新段，段与段之间空一行。
3. 断开位置必须是最接近目标值（默认 250）字符处的以下标点（仅限）：句号、问号、感叹号。
  但以下情况不得把 `.` 作为断开点：英文 `...` 省略号、姓名首字母缩写（如 `M. Foucault`）、常见缩写（如 `Dr.` `Mr.` `e.g.`）、连续大写缩写（如 `U.S.` `U.S.A.`）。
4. 如果段落很长但不存在以上标点，则**不应**分割。
5. 不得在以下内容内部断开：
 - `<sup>...</sup>` 标签包裹内容
 - Markdown 图片标签（如 `![alt](url)`）包裹内容
 - 各种圆括号/方括号包裹内容（如 `(...)`、`（...）`、`[...]`、`【...】`）

## 阶段 3：输出检查
- 所有输出文件在 `translate-typeset/`。
- 输出文件名与输入文件一致。
- 仅断段，不做翻译、纠错或额外排版。
