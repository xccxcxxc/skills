# 标题分类

CRITICAL: 本步骤用于“粗合并”之后、“分割”之前。目标是把 `merge-result/0.rough.md` 的 Markdown 标题整理成可分割、可阅读的层级；不要改正文。

## 输入/输出

- 输入：`merge-result/0.rough.md`
- 输出：原地更新 `merge-result/0.rough.md`
- 建议备份：写入 `merge-result/0.rough.before-heading-classify.md` 或类似文件

## 阶段 1：依据目录确认正文书目

1. 先读取书内目录页，确认正文书目与顺序。
2. 对《先知书》这类结构，正文书目是目录中的各卷书：
   - 以赛亚书、耶利米书、以西结书、何西阿书、约珥书、阿摩司书、俄巴底亚书、约拿书、弥迦书、那鸿书、哈巴谷书、西番雅书、哈该书、撒迦利亚书、玛拉基书
3. 注意区分“导读中提到的书名标题”和“正文实际开始的书名标题”。导读里的书名必须留在导读层级下，不得误升为正文一级标题。

## 阶段 2：标题层级规则

总原则：书名是正文分割单位；章必须从属于书名，不能和书名同级。

- 全书书名、前言、导读、参考书目、译注者简介等前后置大块：`#`
- 导读标题：`##`
- 导读中逐卷说明的书名：`###`
- 正文书名：`#`
- 若该书有“上篇/中篇/下篇”等篇结构：
  - 篇名：`##`
  - 章名：`###`
  - 章内小标题：`####`
- 若该书没有篇结构：
  - 章名：`##`
  - 章内小标题：`###`
- 参考书目内的字母分组：`## A`、`## B` 等

正确示例：

```markdown
# 耶利米書

## 引言

## 一章

### 蒙召
```

```markdown
# 以賽亞書

## 上篇：預言集

### 一章

#### 背叛
```

错误示例：

```markdown
# 耶利米書

# 一章
```

## 阶段 3：清理 OCR 误入正文的页眉页脚标题

识别因 OCR 错误进入正文的页眉页脚标题，并删除这些重复标题；不要删除正文中真实出现的章节标题、书名标题或小标题。

常见判断方式：

- 同一标题在相邻或相近页反复出现，且位置像页首/页尾，而不是正文结构中的正常标题。
- 重复标题内容与当前章节名、书名、卷名或页眉常见内容一致，但后文没有承接为一个新的正文段落。
- 清理时只删除误入正文的重复标题行；不要改动其前后的正文内容。
- 对目录、正文实际开始处、导读中首次介绍的标题要保留，不能仅凭“标题重复”删除。


若用户接下来要成书，完成标题分类后再执行 `references/排版成书.md`。

## 阶段 4：检查

执行后至少检查：

1. `# 耶利米書` 下的 `一章` 是 `## 一章`，不是 `# 一章`。
2. 有篇结构的书，如《以赛亚书》，为 `# 以賽亞書` → `## 上篇：預言集` → `### 一章`。
3. 导读中的书名为 `###`，正文中的书名为 `#`。
4. 没有连续两个以上空行。
