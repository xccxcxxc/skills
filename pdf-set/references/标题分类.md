# 标题分类

CRITICAL: 本步骤用于“粗合并”之后、“排版成书/导出”之前。目标是把 `merge-result/0.rough.md` 的 Markdown 标题整理成可导航、可导出的层级；不要改正文语义。

## 输入/输出
- 输入：`merge-result/0.rough.md`
- 输出：仍写回 `merge-result/0.rough.md`（可另存备份 `0.rough.before-heading-classify.md`）

## 阶段 1：依据目录确认正文书目
1. 先从封面/目录/内容提要确认本书真实结构：单卷小说、多卷、圣经式篇章、论文等。
2. 找出“应进入 EPUB 目录的导航单位”。常见：
   - 单卷武侠/小说：`内容提要`、`序`、各章（如 `二十五`…`四十五`）
   - 多卷书：卷/部/篇，再章
3. 注意区分“导读中提到的书名标题”和“正文实际开始的书名标题”。导读里的书名必须留在导读层级下，不得误升为正文导航标题。

## 阶段 2：标题层级规则（与 EPUB 目录强相关）
Pandoc 导出默认使用：
- `--split-level=1`
- `--toc-depth=1`

因此：
- **所有应出现在 EPUB 目录中的标题，必须是一级标题 `# `**。
- 仅装饰性的书名/卷名若会吞掉整书目录，就不要作为唯一的 body H1。
- 章内小标题才用 `##` / `###`，它们默认不进目录。

### 推荐结构

#### A. 单卷小说 / 下册章节连续编号（重点）
错误（会导致目录只有书名）：
```markdown
# 月狐门人
## 内容提要
## 二十五
## 二十六
```

正确（目录包含全部导航章节）：
```markdown
（台湾）墨龙

# 内容提要
# 二十五
# 二十六
# 四十五
```

规则：
1. 书名优先放 metadata / title page / 封面，不作为唯一 body H1。
2. `内容提要`、序跋、正文各章全部升为 `# `。
3. 删除页眉页脚造成的重复书名标题。
4. 不要把真实章节降成 `##`，否则 `--toc-depth=1` 时目录会丢章。

#### B. 有“书 → 篇/部 → 章”结构
```markdown
# 以賽亞書
## 上篇：預言集
### 一章
```
若目录只需要篇/部，则把篇/部做成 `# `；若目录需要章，则章必须是 `# `。

#### C. 圣经/注疏式
```markdown
# 耶利米書
## 一章
### 蒙召
```
若阅读器目录需要“章”，把 `一章` 升为 `# `。

## 阶段 3：清理 OCR 误入正文的页眉页脚标题
识别并删除误入正文的重复标题；不要删除真实章节标题。

常见信号：
- 同一标题在相邻页反复出现，像页首/页尾。
- 与书名、卷名、当前章名相同，但没有开启新段落结构。
- 封面后重复出现的书名大标题。

## 阶段 4：检查
1. 数一数“应进目录的导航标题”数量；它们必须都是 `# `。
2. 确认没有“整本书只有 1 个 H1 书名，其余章节全是 H2”的结构。
3. 单卷小说常见检查：`内容提要` + 全部章号都在 H1 列表中。
4. 若用户接下来要成书，完成后再执行 `references/排版成书.md`。
