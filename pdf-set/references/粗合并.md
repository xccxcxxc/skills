# 粗合并

输入：`ocr-result/N.md`。输出：`merge-result/0.rough.md`。

```bash
python scripts/merge_rough.py --base-dir "书籍目录"
```

## 合并门禁
默认 fail closed：
- 只接受严格数字文件 `N.md`；
- 页码必须连续；
- 发现 `*.fail.json` 即失败；
- 每页 GFM/HTML 表结构必须有效；
- 非数字 Markdown、fail/partial 文件不参与合并。

`--allow-missing` 只用于明确授权的调试，不可用于最终 EPUB。

## 跨页表
默认执行保守合并：
- 相邻页边缘表格表头 signature 完全一致才合并；
- GFM：删除续页重复 header/separator，追加数据行；
- HTML：保留一个 `<thead>`，合并 `<tbody>`；
- 列结构不一致时不自动合并。

可用 `--no-merge-continued-tables` 禁用。报告写入：

```text
merge-result/0.rough.md.report.json
```

## 图片占位
- 合并前应先跑 `crop_figures.py`（`pdf_to_epub.py` 会自动跑）。
- `🀄️figures/N-1.jpg` / `🀄️N.jpg` 会物化为 `![](assets/...)`。
- 优先使用裁切图；资源缺失则合并失败（或 `--allow-missing` 仅调试）。

合并只压缩多余空行，不做纠错、删字或语义改写。
