## 翻译脚本说明

- 脚本：`scripts/translation.py`
- Prompt：默认读取 `assets/translate_prompt.md`
- 翻译统一使用 OpenAI Python 库。
- 密钥配置：读取 `assets/secrets_openai.txt`（`base_url`、`api_key`、`model`）

## 输入/输出

- 输入：`translate-typeset/` 中按序号命名的 Markdown 文件（如 `0.md`、`1.md`）
- 输出：`translate-result/` 中同名文件（如 `0.md`、`1.md`）

默认路径（不传参数时）：
- 输入：`<当前目录>/translate-typeset`
- 输出：`<当前目录>/translate-result`

## 自动续翻逻辑

在未指定 `--start/--end` 时：
1. 扫描输出目录，找到序号最大的输出文件。
2. 检查该文件的输出段落数（按 `---` 分隔符计数）。
3. 对比输入目录同名文件总段落数（按空行分段）。
4. 若输出段落更少：从断点继续翻译。
5. 若输出段落不少：从下一序号文件开始翻译。

## 进度显示

- 双进度条：
  - `Files`：当前已完成文件数/总文件数
  - `Paras`：当前文件已完成段落数/输入总段落数
- 每次发送 API 前会在进度条上方显示当前段落预览。
- 开始处理首个文件前会输出该文件总段落数与前 5 段预览。

## API 输入输出

- 每次请求默认发送 5 段（可用 `--chunk-size` 调整）。
- 输入给 API 时仅发送段落正文（不包含 `Para i` 或 `---` 标签）。
- API 返回文本会按顺序直接追加写入输出文件。

## 参数

- `--base-dir`：书籍根目录（默认当前目录）
- `--book-name`：书名子目录（拼接为 `<base-dir>/<book-name>`）
- `--base-dir-from`：从文本文件读取 `base-dir`
- `--input-dir`：输入目录
- `--input-dir-from`：从文本文件读取输入目录
- `--output-dir`：输出目录
- `--output-dir-from`：从文本文件读取输出目录
- `--start`：起始序号（含）
- `--end`：结束序号（含）
- `--chunk-size`：每次请求段落数（默认 5）
- `--prompt-file`：Prompt 文件路径
- `--prompt-file-from`：从文本文件读取 Prompt 文件路径

## 示例
优先使用此方法
```bash
python scripts/translation.py --base-dir "C:\Git\translation" --book-name "S17"
```

```bash
python scripts/translation.py --input-dir "C:\Git\translation\S17\translate-split" --output-dir "C:\Git\translation\S17\translate-result" --start 12 --end 20
```
